## 1. 问题陈述

系统架构设计师（软考高级，代码 038）考试自 2023 年启用第二版教材，近 15 年首次重大改版，旧资料失效。考试三科（综合知识 75 题选择题 / 案例分析 5 选 4 / 论文 4 选 1），三科均需 ≥45 分通过。备考痛点：教材 20 章体量大、知识点分散、案例与论文需结构化训练、缺乏个性化薄弱点反馈。

**目标状态**：构建一套备考系统，支持邮件验证码登录与跨端同步，覆盖学习、习题、写作指导、模拟考四大功能。

## 2. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|---------|------|---------|
| 备考者（本人） | 系统架构设计师考生，软件工程背景 | 高效记忆考点、针对性练习、论文结构化训练、全真模拟 |

单一备考者，支持邮箱验证码登录，无密码；数据按用户隔离，可跨端同步。

## 3. v1 功能边界

### 3.1 v1 必做

| 功能 | 解决什么 | 验收标准 |
|------|----------|----------|
| 知识点学习（20章） | 知识点分散无体系 | 按教材章节树形浏览，标记考试权重 |
| 间隔重复复习 | 遗忘曲线导致记忆低效 | SM-2 算法调度，连续 7 天队列正确 |
| 选择题练习 | 缺乏针对性练习 | 章节/随机/错题三模式，自动评分 |
| 错题本+统计分析 | 不知薄弱点在哪 | 自动收录错题，知识点正确率统计 |
| AI 智能选题 | 题库量不足 | 薄弱点（正确率<60%）触发 AI 智能选题，从题库筛选相关题目 |
| 论文模板+范文 | 论文无结构化训练 | 10大高频主题模板+范文，5维度框架 |
| AI 论文评分 | 无写作反馈 | 5维度评分+逐段点评+改进建议 |
| 模拟考（三科） | 无法全真模拟 | 综合知识75题/案例5选4/论文4选1，计时+评分 |
| 薄弱点推荐 | 不知该学什么 | 数据驱动（正确率+模考成绩）推荐复习 |
| 深色模式+移动适配 | 随时随地复习 | 桌面+移动自适应，主题切换 |
| 历年真题导入 | 真实题目来源 | 支持从 GitHub 资源导入 2009-2025 真题 |

### 3.2 v1 推迟

| 功能 | 推迟理由 | 何时再评估 |
|------|----------|------------|
| 多软考科目扩展 | 当前聚焦系统架构设计师 | 本科通过后 |

| 社交/分享功能 | 非社交场景，注册仅为跨端同步 | 无需求 |

| 方向 | 不做原因 |
|------|----------|
| 社交功能 | 注册仅为跨端同步，非社交场景 |
| 视频课程/直播 | 非文本学习场景 |
| 题库在线编辑后台 | 静态文件 Git 维护 |

## 4. 核心场景

### 场景一：日常学习+复习

用户打开系统 → 首页显示今日复习队列（SM-2 调度） → 逐张复习知识卡片，评分掌握度（0-5） → 系统更新下次复习日期 → 完成后查看学习仪表盘（连续天数、章节掌握度）

### 场景二：章节练习+薄弱点识别

用户选择第7章（架构设计） → 做 20 道选择题 → 提交查看答案+解析 → 错题自动入册 → 统计该章节正确率 → 若正确率<60%，AI 生成 3-5 道补充题 → 推荐至复习队列

### 场景三：论文写作+AI评分

用户选择"论微服务架构" → 查看模板+范文 → 在写作工作台分节撰写（摘要/背景/方案/效果/结论） → 实时字数统计 → 提交 AI 评分 → 获得5维度得分+逐段点评+改进建议

### 场景四：全真模拟考

用户启动综合知识模拟考 → 75题计时150min → 提交自动评分（对标45/75合格线）→ 查看章节得分分布+错题列表 → 成绩记录趋势分析

## 5. 验收标准

- [ ] 20章知识点可按教材结构浏览，标记考试权重
- [ ] SM-2 间隔重复队列连续 7 天正确调度
- [ ] 三种练习模式（章节/随机/错题）正常出题
- [ ] 错题自动收录，统计分析准确
- [ ] 薄弱点（正确率<60%）触发 AI 出题
- [ ] 论文模板+10大主题范文齐全
- [ ] AI 论文 5 维度评分可用
- [ ] 三科模拟考计时+评分正常
- [ ] 数据驱动薄弱点推荐合理
- [ ] 深色模式+移动端核心功能可用
- [ ] 历年真题可导入

## 6. 非功能需求

| 维度 | 需求 | 备注 |
|------|------|------|
| 性能 | 页面首屏 <2s，AI 响应 <30s | 含超时重试 |
| 可用性 | 本地部署，AI 降级时核心功能不受影响 | 无 SLA |
| 兼容性 | Chrome/Safari/Firefox + iOS/Android | 现代浏览器 |
| 安全 | API Key 加密存储，JWT 鉴权，数据按 user_id 隔离 | 有敏感数据（API Key） |
| 可维护性 | 题库/知识点 Git 维护，AI Prompt 集中配置 | 静态数据 |

### 2.2 任务详情

#### T001: vite-plus 项目初始化（React + Elysia + SQLite）

**任务描述**：搭建 vite-plus 全栈项目，包含前端 React 与后端 Elysia 基础骨架，建立统一的开发、构建与 lint 流程。

**验收标准**：
- [ ] 项目目录结构与 workspace 划分正确
- [ ] 前端 dev server 可启动
- [ ] 后端 dev server 可启动
- [ ] 共享类型包可正常引用

#### T002: SQLite 数据库 schema 初始化（全部表）

**任务描述**：创建 SQLite 数据库初始化脚本，包含用户、AI 配置、知识卡片、题库、答题记录、错题本、论文草稿等全部表结构。

**验收标准**：
- [ ] 所有表结构符合 PRD 数据隔离要求
- [ ] 用户表、AI 配置表、知识卡片表、答题记录表、错题表、论文草稿表均已创建
- [ ] schema 迁移脚本可重复执行

#### T003: 邮件验证码登录（SMTP + JWT + refresh token）

**任务描述**：实现邮箱验证码发送、校验、JWT 签发、refresh token 机制，首次登录自动注册。

**验收标准**：
- [ ] 可发送 6 位验证码邮件，有效期 5 分钟
- [ ] 60s 内不可重复发送
- [ ] 验证通过后签发 JWT（7 天）和 refresh token（30 天）
- [ ] 数据按 user_id 隔离

#### T004: 退出登录

**任务描述**：实现前端清除 token 并返回登录页，后端可选将 token 加入黑名单。

**验收标准**：
- [ ] 前端清除 JWT 与 refresh token
- [ ] 退出后跳转登录页
- [ ] 后端鉴权拒绝已退出 token

#### T005: AI 配置（多模型 + API Key 加密存储）

**任务描述**：实现用户级 AI provider 配置，支持 OpenAI/Anthropic/DeepSeek/MiniMax/Kimi，API Key 使用 AES-256 加密存储，前端不接触 Key。

**验收标准**：
- [ ] 可配置 provider、模型、API Key、自定义 base URL
- [ ] API Key 加密存储，解密仅在后端
- [ ] 提供测试连接功能验证 Key 有效性

#### T006: 主题与响应式（深色/浅色 + 移动自适应）

**任务描述**：实现深色/浅色主题切换，确保移动端核心功能可用。

**验收标准**：
- [ ] 主题切换生效并持久化
- [ ] 移动端布局无横向滚动、核心功能可访问

#### T007: cloudflared 隧道 + CORS + HTTPS 配置

**任务描述**：配置 cloudflared 隧道，开放外网 HTTPS 访问，并配置 CORS 仅允许已知来源。

**验收标准**：
- [ ] cloudflared 隧道可建立并稳定运行
- [ ] 外网可通过 HTTPS 访问服务
- [ ] CORS 仅允许已知来源

#### T008: SMTP 邮件服务配置

**任务描述**：配置 SMTP 邮件服务，确保验证码邮件可正常发送。

**验收标准**：
- [ ] SMTP 服务可发送邮件
- [ ] 邮件到达收件箱而非垃圾箱

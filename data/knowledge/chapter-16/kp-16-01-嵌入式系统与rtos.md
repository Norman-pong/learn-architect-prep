## 概述
嵌入式系统（Embedded System）是为了特定应用而专门构建的计算机系统，其软硬件架构设计与通用计算机存在显著差异[^1]。RTOS（Real-Time Operating System，实时操作系统）是面向控制、通信等强实时领域的关键基础软件，能够提供强制性任务调度与可预测的时间响应，是嵌入式系统架构设计的核心支撑[^2]。

## 核心知识点

### 1. 嵌入式系统基本特征与硬件组成
嵌入式系统具有“专用、资源受限、强实时、软硬件协同”的典型特征。教材将其发展划分为五个阶段：单片微型计算机（SCM）阶段、微控制器（MCU）阶段、片上系统（SoC）阶段、以 Internet 为基础的嵌入式系统阶段，以及智能化与云技术推动阶段[^3]。其硬件通常由嵌入式微处理器/控制器、存储器（RAM/ROM）、内/外总线、定时/计数器、看门狗、I/O 接口及外部设备组成[^4]。

| 处理器类型 | 英文缩写 | 主要特点 |
|------------|----------|----------|
| 微处理器 | MPU | 体积小、成本低、可靠性高，但板级器件多、保密性较差[^5] |
| 微控制器 | MCU | 单片化，体积小、功耗低、成本低、可靠性高，又称单片机[^5] |
| 信号处理器 | DSP | 哈佛结构，适合 DSP 算法，执行速度快、编译效率高[^5] |
| 图形处理器 | GPU | 擅长 3D 图形渲染，可减轻 CPU 负担[^5] |
| 片上系统 | SoC | 将处理器内核与外设集成，可基于标准库设计，集成度高[^5] |

存储器按使用类型分为 RAM 与 ROM。RAM 可读写，但断电后数据丢失；ROM 信息硬接线固化，正常工作时不可修改，可进一步细分为 MASK ROM、PROM、EPROM 等[^6]。

### 2. 嵌入式操作系统架构与特点
嵌入式操作系统（EOS）负责嵌入式系统全部软硬件资源分配、任务调度、控制与协调并行活动[^7]。与通用操作系统相比，其典型特点包括：可剪裁性、可移植性、强实时性、强紧凑性、高质量代码、强定制性、标准接口、强稳定性与弱交互性、强确定性、操作简洁、较强的硬件适应性和可固化性等[^8]。

嵌入式操作系统通常分为两类：一是面向控制、通信等领域的嵌入式实时操作系统，如 VxWorks、Nucleus 等；二是面向消费电子产品的非实时嵌入式操作系统，如 Android、iOS、WinCE 等[^9]。从内核架构看，可分为宏内核（Monolithic Kernel）与微内核（Micro Kernel）。宏内核中用户服务与内核服务在同一空间实现，代码耦合度高，如 VxWorks 5.5、VRTX；微内核中用户服务与内核服务在不同地址空间实现，通过消息传递通信，结构清晰、移植性好、伸缩性强，但性能偏低，如嵌入式 Linux、L4、WinCE[^10]。

### 3. 实时操作系统任务调度
RTOS 的任务状态可分为执行态、就绪态和阻塞态。执行态指任务正在 CPU 上运行；就绪态指任务已分配到除 CPU 外的必要资源，等待 CPU 即可执行；阻塞态指任务因等待某事件而暂时无法执行[^11]。调度算法可从三个维度划分：

- **离线/在线调度**：离线调度信息在运行前确定，具有确定性；在线调度信息在运行中动态获取，灵活性高[^12]。
- **抢占/非抢占调度**：抢占式允许高优先级任务打断当前任务；非抢占式一旦任务开始运行，只能主动放弃或阻塞[^12]。
- **静态/动态调度**：静态调度中任务优先级设计时确定、运行中不变，如 RMS；动态调度中优先级在运行中确定并可能变化，如 EDF[^12]。

典型强实时调度算法包括：

- **EDF（最早截止时间优先）**：按任务截止时间早晚确定优先级，截止时间越早优先级越高[^13]。
- **LLF（最低松弛度优先）**：按任务松弛度（必须完成时间−本身运行时间−当前时间）确定优先级，松弛度越低越优先[^13]。
- **RMS（单调速率调度）**：静态优先级调度算法，任务周期越短优先级越高，被认为是周期性任务最优静态调度算法[^13]。

RTOS 的核心价值在于：通过强制性的任务调度机制，为关键任务提供可预测的响应时间，满足系统时间约束[^2]。

### 4. 嵌入式基础软件与架构方法
嵌入式基础软件包括嵌入式操作系统、嵌入式数据库、嵌入式中间件等。嵌入式数据库与数据库服务器最大的区别在于运行地址空间：嵌入式数据库与应用程序运行在同一进程，而数据库服务器以独立守护进程运行[^14]。常见嵌入式数据库产品有 SQLite、Berkeley DB、eXtremeDB、Firebird 嵌入式服务器版等[^15]。

嵌入式系统软件架构设计常用两种方法：一是 **ABSD（基于架构的软件设计）**，强调自顶向下、递归细化，由业务、质量和功能需求驱动架构设计[^16]；二是 **ADD（属性驱动的软件设计）**，以一组质量属性场景为输入，通过架构模式和战术递归分解系统，满足可靠性、安全性、可用性、可修改性、性能等质量属性[^17]。

## 考点与易错点

1. **RTOS 的核心特征**：考试常把 RTOS 与通用操作系统混淆。RTOS 最突出的特点是“强制性任务调度 + 可预测响应时间”，而非图形界面、多用户登录或大容量外部存储[^18]。
2. **宏内核 vs 微内核**：宏内核性能高但耦合度高；微内核结构清晰、可移植性强但性能偏低。需根据应用场景判断选型[^10]。
3. **调度算法选择**：RMS 是静态优先级、周期越短优先级越高；EDF 按截止时间动态排序；LLF 按松弛度排序。三者适用场景不同，易在“静态/动态”维度上出错[^12][^13]。
4. **嵌入式数据库 vs 数据库服务器**：嵌入式数据库与应用程序同进程，无需独立安装部署；数据库服务器以 C/S 模式独立运行，二者地址空间不同[^14]。

## 真题示例

**56. 嵌入式系统中，RTOS 的主要特点是（  ）。**[^19]

A. 强制性任务调度和可预测响应时间  
B. 支持多用户同时登录  
C. 提供图形用户界面  
D. 支持大容量外部存储  

**答案：A**

**解析**：RTOS 强调实时性，通过优先级抢占、时间片轮转等机制提供强制性任务调度，并保证关键任务在截止时限前完成，因此其核心特点是“强制性任务调度和可预测响应时间”。选项 B、C、D 属于通用操作系统或消费级操作系统的特征，不是 RTOS 的主要特点[^18][^19]。

## 记忆要点

- **嵌入式系统**：专用、受限、实时、协同。
- **RTOS 三要素**：强制调度、可预测响应、截止时限保障。
- **调度算法口诀**：
  - 周期短优先 → RMS（静态）；
  - 截止早优先 → EDF（动态）；
  - 松弛小优先 → LLF（抢占）。
- **内核对比**：宏内核性能高、耦合高；微内核清晰、可移植、性能低。
- **嵌入式数据库**：与应用同进程，体积小、免部署。

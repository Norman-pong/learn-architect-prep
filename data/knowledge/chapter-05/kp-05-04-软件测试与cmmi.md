## 概述

软件测试是使用人工或自动手段运行或测定软件系统的过程，目的在于检验其是否满足规定需求或弄清预期结果与实际结果的差别[^1]。测试应贯穿整个软件开发过程，而非仅在编码完成后进行。CMMI（能力成熟度模型集成）则是评估和改进软件开发过程能力的框架，将软件过程改进组织成5个成熟度等级[^2]。

## 核心知识点

### 测试方法分类

软件测试方法可从多个维度分类[^1]：

| 分类维度 | 类型 | 说明 |
|:---:|:---|:---|
| 程序执行状态 | 静态测试（ST） | 被测程序不运行，依靠分析检查源程序 |
| | 动态测试（DT） | 运行被测试程序，比较运行结果与预期结果 |
| 内部结构了解程度 | 黑盒测试 | 不考虑程序内部结构，根据需求规格设计测试用例 |
| | 白盒测试 | 借助程序内部逻辑，检测内部动作是否按设计进行 |
| | 灰盒测试 | 介于黑盒与白盒之间，重视输出正确性也看重内部逻辑 |
| 执行方式 | 人工测试（MT） | 由测试人员手动执行 |
| | 自动化测试（AT） | 以机器执行预先设定的测试条件 |

### 白盒测试覆盖标准

白盒测试根据测试用例的覆盖程度，分为以下级别（由弱到强）[^1]：
1. **语句覆盖**：每条语句至少执行一次
2. **判定覆盖（分支覆盖）**：每个判定的真假分支至少执行一次
3. **条件覆盖**：每个条件的各种可能值至少满足一次
4. **路径覆盖**：每条可能的路径至少执行一次（最强覆盖标准）

**McCabe圈复杂度**：用于衡量程序复杂度，计算公式为 **V(G) = E - N + 2P**，其中E为边数，N为节点数，P为连通分量数（通常为1）。流图中有n个判断节点时，圈复杂度为 **n + 1**[^3]。

### 测试阶段

从阶段上划分，软件测试包括[^4]：
- **单元测试**：对软件模块进行测试，可采用静态分析+黑盒/白盒结合
- **集成测试**：对组装后的模块进行测试，验证接口和结构组装的正确性
- **系统测试**：采用黑盒测试，检查系统是否符合需求；包括功能、性能、健壮性、压力、可靠性等测试
- **验收测试**：软件正式交付前的测试，由用户执行；通过后进行Alpha测试或Beta测试

### CMMI五级模型

CMMI将软件过程改进组织成5个成熟度等级，共包括18个关键过程域[^2]：

| 级别 | 名称 | 特征 |
|:---:|:---|:---|
| 1 | **初始级** | 过程随意混乱，成功依赖个人能力与英雄主义 |
| 2 | **已管理级** | 确保项目级过程被策划、文档化、执行、监督和控制 |
| 3 | **已定义级** | 企业定义适合自身的标准流程并制度化，开始项目积累 |
| 4 | **量化管理级** | 建立产品质量、服务质量和过程性能的定量目标；过程性能可预测 |
| 5 | **优化级** | 通过增量式与创新式的过程与技术改进，持续改进过程性能 |

**关键区别**：3级与4级的关键区别在于对过程性能的可预测性；5级关注使用多项目数据对组织级绩效进行关注[^2]。

### 净室软件工程（CSE）

净室软件工程是一种应用数学与统计学理论以经济方式生产高质量软件的工程技术[^5]：
- **理论基础**：函数理论（完备性、一致性、正确性）和抽样理论
- **技术手段**：统计过程控制下的增量式开发、基于函数的规范与设计（盒子结构）、正确性验证、统计测试
- **核心思想**：在规约和设计中消除错误，以"净"的方式制作，避免成本高昂的错误消除过程

## 考点与易错点

1. **白盒覆盖标准强度**：语句覆盖 < 判定覆盖 < 条件覆盖 < 路径覆盖。路径覆盖是最强的覆盖标准[^1]。
2. **CMMI级别对应**：2级=已管理级，3级=已定义级，4级=量化管理级，5级=优化级。注意"定量管理级"对应的是**第4级**[^2]。
3. **McCabe圈复杂度**：V(G) = E - N + 2P。若流图有6个判断节点，则V(G) = 6 + 1 = 7[^3]。
4. **Alpha测试 vs Beta测试**：Alpha测试在开发环境下由用户进行；Beta测试在实际环境中由多个用户进行[^4]。

## 真题示例

**题1**（对应Q17）CMMI级别中，"定量管理级"对应第几级？（  ）
A. 2   B. 3   C. 4   D. 5

**答案：C**

**解析**：CMMI五级模型中，第4级为量化管理级（Quantitatively Managed），组织建立了产品质量、服务质量以及过程性能的定量目标，过程性能具有可预测性[^2]。

**题2**（对应Q18）白盒测试中，最强的覆盖标准是（  ）。
A. 语句覆盖   B. 判定覆盖   C. 条件覆盖   D. 路径覆盖

**答案：D**

**解析**：白盒测试的覆盖标准由弱到强依次为：语句覆盖→判定覆盖→条件覆盖→路径覆盖。路径覆盖要求每条可能的路径至少执行一次，是最强的覆盖标准[^1]。

**题3**（对应Q20）某程序的流图有6个判断节点，其McCabe圈复杂度为（  ）。
A. 5   B. 6   C. 7   D. 8

**答案：C**

**解析**：McCabe圈复杂度V(G) = 判断节点数 + 1。流图有6个判断节点时，V(G) = 6 + 1 = 7[^3]。

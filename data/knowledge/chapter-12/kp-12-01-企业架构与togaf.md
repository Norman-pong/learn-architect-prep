## 概述

信息系统架构（Information System Architecture，ISA）是反映政府、企业或事业单位信息系统的各组成部分之间，以及信息系统与相关业务、技术之间关系的体系结构[^1]。企业架构框架（如 Zachman、FEAF、TOGAF）为组织级信息系统规划提供结构化方法，其中 TOGAF 及其架构开发方法（ADM）是业界使用最广泛的企业架构框架[^2]。本章以 ADM 为核心，说明如何迭代地开发业务、数据、应用和技术架构，并据此制订迁移与治理方案。

## 核心知识点

### 1. 信息系统架构的定义与分类

信息系统架构是软件系统结构、行为和属性的高级抽象，由构成元素、元素的外部可见属性及其关系组成[^3]。教材将其概括为：
- 概念层次：艺术、科学、方法与建设风格；
- 物理层次：经过架构工作后产生的物理结构及其相互作用的结果[^4]。

从物理结构看，信息系统分为集中式与分布式两大类；分布式结构又可分为一般分布式与客户机/服务器（C/S）模式[^5]。从逻辑结构看，信息系统可分解为各职能子系统，并可通过横向综合、纵向综合和纵横综合三种方式实现子系统整合[^6]。

### 2. TOGAF 框架的组成与核心思想

TOGAF（The Open Group Architecture Framework）由 The Open Group 制定，1995 年发布架构框架，其基础源自美国国防部信息管理技术架构（TAFIM）[^7]。TOGAF 9 版本包含六个组件[^8]：

| 组件 | 说明 |
| --- | --- |
| 架构开发方法（ADM） | TOGAF 核心，描述开发企业架构的分步方法 |
| ADM 指南和技术 | 应用 ADM 的指南和技术 |
| 架构内容框架 | 架构工件的结构化元模型、可重用架构构建块（ABB）及典型可交付成果 |
| 企业连续体和工具 | 对架构活动输出进行分类和存储的分类法与工具 |
| TOGAF 参考模型 | 技术参考模型（TRM）和集成信息基础设施参考模型（III-RM） |
| 架构能力框架 | 企业内建立和运营架构实践所需的组织、流程、技能、角色和职责 |

TOGAF 的核心思想包括：模块化架构、内容框架、扩展指南和架构风格[^9]。

### 3. ADM 架构开发方法

架构开发方法（Architecture Development Method，ADM）是 TOGAF 规范中最为核心的内容，定义了开发企业架构所需执行的步骤及关系[^10]。ADM 全生命周期分为 10 个阶段，这些阶段反复迭代[^11]：

| 阶段 | 名称 | 主要活动 |
| --- | --- | --- |
| 预备阶段 | 准备 | 定义组织、架构框架、架构原则和工具 |
| 阶段 A | 架构愿景 | 设定范围、约束和期望，创建架构愿景，取得批准 |
| 阶段 B | 业务架构 | 开发基线与目标业务架构，执行差距分析 |
| 阶段 C | 信息系统架构（应用与数据） | 开发数据架构和应用架构，完成差距分析 |
| 阶段 D | 技术架构 | 完成基础设施设计，定义物理实现 |
| 阶段 E | 机会及解决方案 | 确定主要实施项目，对项目分组并纳入过渡架构 |
| 阶段 F | 迁移规划 | 绩效分析和风险评估，制订详细实施和迁移计划 |
| 阶段 G | 实施治理 | 定义架构约束，提供架构监督，发布架构合同 |
| 阶段 H | 架构变更管理 | 持续监测和变更管理，使架构价值最大化 |
| 需求管理 | 跨阶段 | 识别、存储、管理需求，贯穿 ADM 所有阶段 |

ADM 的三个迭代级别包括：基于 ADM 整体的迭代、多个开发阶段间的迭代，以及一个阶段内部的迭代[^12]。需求管理是 ADM 流程的中心，处理需求变化的能力对 ADM 过程至关重要[^13]。

### 4. 阶段 H：架构变更管理

阶段 H 是 ADM 的最后一个阶段，提供持续监测和变更管理流程，确保架构可以响应企业需求并将架构对于业务的价值最大化[^14]。其关键目标包括：确保基线架构持续符合当前实际情况；评估架构性能并提出改进建议；管理风险；为实施治理阶段建立新的企业架构基线建立架构变更管理流程[^15]。

## 考点与易错点

1. **ADM 阶段名称与顺序**：10 个阶段顺序常以选择题形式考查，特别是阶段 H（架构变更管理）容易被混淆为“迁移规划”或“需求管理”[^16]。
2. **需求管理的位置**：需求管理不是 ADM 的“第 11 阶段”，而是跨所有阶段的动态过程，是 ADM 流程的中心[^17]。
3. **TOGAF 的六个组件**：需区分“架构开发方法”与“架构能力框架”，前者是开发方法，后者是组织、流程、技能、角色和职责[^18]。

## 真题示例

**题目**：TOGAF 架构开发方法（ADM）的“H”阶段是（  ）。

A. 架构变更管理   
B. 迁移规划   
C. 机会及解决方案   
D. 需求管理

**答案**：A

**解析**：ADM 的 H 阶段为架构变更管理（Architecture Change Management），负责持续监测和变更管理，确保架构响应企业需求并最大化架构对业务的价值。B 选项“迁移规划”对应阶段 F，C 选项“机会及解决方案”对应阶段 E，D 选项“需求管理”是贯穿 ADM 各阶段的动态过程，而非 H 阶段[^19]。

## 记忆要点

- **ADM 十阶段口诀**：预愿业务信（应用+数据）技，机迁管变需；
  - 预备阶段 → 架构愿景 → 业务架构 → 信息系统架构（应用/数据） → 技术架构 → 机会及解决方案 → 迁移规划 → 实施治理 → 架构变更管理（H） → 需求管理（贯穿）。
- **阶段 H = 变更管理**：H 是“换（Change）”，不是“迁（Move）”。
- **TOGAF 六组件**：ADM 指南、内容、连续体、参考模型、能力框架，核心在 ADM。
